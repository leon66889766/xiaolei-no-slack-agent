# 产品需求文档（PRD）

## 小雷没摸鱼 Agent — 全栈对话式资源管理平台

---

## 1. 项目信息

| 项目 | 内容 |
|------|------|
| **项目名称** | `xiaolei-no-slack-agent` |
| **产品名称** | 小雷没摸鱼 Agent |
| **文档版本** | v1.0 |
| **语言** | 中文 |
| **建议技术栈** | Next.js (React) 全栈方案 — 便于一体化部署，最终选型由架构师决定 |
| **原始需求复述** | 构建一个全栈对话式资源管理平台，包含无需登录的前台对话式 AI 交互界面（支持多格式资源预览/下载）和需登录的后台管理界面（支持资源上传、分类、标签、搜索、删除）。AI 对话逻辑需精准匹配用户意图并从资源库调取对应内容。要求前后端功能完整、可部署上线。 |

---

## 2. 产品定义

### 2.1 产品目标

| 编号 | 产品目标 | 说明 |
|------|----------|------|
| G1 | **打造"对话即检索"的零门槛资源获取体验** | 用户无需了解资源分类体系，通过自然语言对话即可精准获取所需资源，降低资源查找的认知成本 |
| G2 | **建立统一的多格式资源管理中心** | 为管理员提供一站式的资源上传、分类、标签、检索与管理能力，支持图片/视频/文档/链接等多种格式集中管理 |
| G3 | **实现从开发到上线的完整可交付产品** | 前后端功能完整、前后台数据打通，完成部署上线，确保平台可公开访问与稳定运行 |

### 2.2 用户故事

#### 前台用户（访客）

| 编号 | 用户故事 |
|------|----------|
| US1 | 作为一个**访客**，我想要**无需注册登录即可使用对话界面**，这样我能快速获取所需资源而不被登录流程阻挡。 |
| US2 | 作为一个**访客**，我想要**用自然语言描述我的需求**（如"给我一张科技感的背景图"），这样 AI 能自动从资源库检索并返回匹配结果。 |
| US3 | 作为一个**访客**，我想要**在线预览图片/视频/文档**，这样我能在下载前确认资源内容是否符合需求。 |
| US4 | 作为一个**访客**，我想要**一键下载所需资源**，这样我能快速获取文件用于自己的工作。 |

#### 后台管理员

| 编号 | 用户故事 |
|------|----------|
| US5 | 作为一个**管理员**，我想要**通过账号密码安全登录后台**，这样只有授权人员才能管理资源库。 |
| US6 | 作为一个**管理员**，我想要**批量上传多种格式的资源文件**（图片/视频/文档等），这样我能高效扩充资源库。 |
| US7 | 作为一个**管理员**，我想要**手动添加外部链接资源**，这样我能将网络资源纳入统一管理体系。 |
| US8 | 作为一个**管理员**，我想要**对资源进行分类和打标签**，这样资源库结构清晰、便于 AI 精准检索。 |
| US9 | 作为一个**管理员**，我想要**搜索和删除资源**，这样我能维护资源库的时效性和准确性。 |

## 3. 技术规范

### 3.1 需求池（Requirements Pool）

#### P0 — Must Have（核心功能，必须完成）

| 编号 | 功能模块 | 需求描述 | 验收标准 |
|------|----------|----------|----------|
| P0-1 | 前台对话界面 | 提供对话式 AI 交互界面，无需登录即可使用 | 用户打开页面即可输入文字发起对话，界面响应时间 < 3s |
| P0-2 | AI 意图识别与资源检索 | AI 解析用户自然语言意图，从资源库中检索匹配资源并返回 | 对常见关键词意图识别准确率 ≥ 80%，返回结果与资源库实际数据一致 |
| P0-3 | 资源预览展示 | 在对话中展示检索结果，支持图片/视频/文档/链接的预览 | 图片可查看、视频可播放、文档可在线预览、链接可点击跳转 |
| P0-4 | 资源下载 | 用户可一键下载检索到的资源文件 | 点击下载后获取原始文件，文件完整无损 |
| P0-5 | 后台登录认证 | 管理员通过账号密码登录后台管理系统 | 错误密码被拒绝，正确密码进入后台；未登录访问后台被拦截 |
| P0-6 | 资源上传 | 支持上传图片（JPG/PNG/SVG）、视频（MP4）、文档（Word/PDF/Excel）等多格式文件 | 上传成功后文件可存储、可被前台检索到；支持单个及多文件上传 |
| P0-7 | 外部链接资源添加 | 支持手动添加外部链接作为资源条目 | 可填写链接标题、URL、描述、分类标签；添加后可被前台检索 |
| P0-8 | 资源删除 | 管理员可删除资源库中的资源 | 删除后资源从前台检索结果中消失，文件从存储中移除 |
| P0-9 | 前后端数据打通 | 前台能正确查询后台录入的资源数据 | 后台上传的资源，前台对话可检索到并正确展示 |
| P0-10 | 部署上线 | 完成开发后部署到公开可访问的平台 | 提供可公开访问的 URL，前后台均正常运作 |

#### P1 — Should Have（重要功能，尽量完成）

| 编号 | 功能模块 | 需求描述 | 验收标准 |
|------|----------|----------|----------|
| P1-1 | 资源分类管理 | 支持创建/编辑/删除资源分类（如图片、视频、文档、链接等大类） | 分类可增删改；资源可归入指定分类；前台可按分类筛选 |
| P1-2 | 标签管理 | 支持为资源添加多个标签，支持标签的创建与删除 | 一个资源可有多个标签；支持按标签搜索资源 |
| P1-3 | 后台资源搜索 | 管理员可在后台按关键词、分类、标签搜索资源 | 搜索结果准确，支持组合筛选条件 |
| P1-4 | 资源列表与分页 | 后台以列表/网格形式展示所有资源，支持分页 | 资源列表展示缩略图、名称、格式、分类、标签、上传时间；分页正常 |
| P1-5 | 对话历史 | 前台保留当前会话的对话历史 | 用户可查看本轮对话的上下文，支持清空重新开始 |
| P1-6 | 资源元信息展示 | 展示资源的文件大小、格式、上传时间等元信息 | 元信息准确展示在预览卡片上 |

#### P2 — Nice to Have（增强体验，时间允许时实现）

| 编号 | 功能模块 | 需求描述 | 验收标准 |
|------|----------|----------|----------|
| P2-1 | 多轮对话上下文 | AI 支持基于上下文的多轮对话，理解"再找几个类似的"等指代 | 连续追问时能理解上下文并调整检索结果 |
| P2-2 | 资源推荐 | 无明确需求时，AI 主动推荐热门/最新资源 | 空输入或模糊输入时返回推荐列表 |
| P2-3 | 上传进度与批量操作 | 上传时显示进度条，支持批量删除资源 | 进度条实时更新；批量删除有确认提示 |
| P2-4 | 资源统计仪表盘 | 后台首页展示资源总数、分类占比、近期上传趋势等统计 | 图表数据准确，实时反映资源库状态 |
| P2-5 | 访问量统计 | 记录资源的检索/下载次数 | 可查看热门资源排行 |
| P2-6 | 暗色模式 | 前后台支持深色/浅色主题切换 | 主题切换流畅，两种模式下内容可读 |

### 3.2 UI 设计稿描述

#### 3.2.1 前台（用户访问界面）

**整体风格**：大气美观，现代感设计，主色调建议采用科技蓝/紫渐变，留白充足，圆角卡片式布局。

**页面 1：对话主页（核心页面）**

```
┌─────────────────────────────────────────────────┐
│  [Logo] 小雷没摸鱼 Agent          [主题切换]      │  ← 顶部导航栏
├─────────────────────────────────────────────────┤
│                                                   │
│         欢迎语 + 简介 + 示例问题引导               │  ← 空状态引导区
│                                                   │
├─────────────────────────────────────────────────┤
│                                                   │
│  [用户气泡] 给我一张科技感的背景图                 │  ← 对话消息区
│                                                   │
│  [AI 气泡] 为你找到以下资源：                      │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐             │
│  │ 缩略图1  │ │ 缩略图2  │ │ 缩略图3  │  ← 资源卡片│
│  │ 名称     │ │ 名称     │ │ 名称     │            │
│  │ 格式·大小│ │ 格式·大小│ │ 格式·大小│            │
│  │[预览][下载]│ │[预览][下载]│ │[预览][下载]│       │
│  └─────────┘ └─────────┘ └─────────┘             │
│                                                   │
├─────────────────────────────────────────────────┤
│  [输入框............................] [发送 ➤]   │  ← 底部输入栏
└─────────────────────────────────────────────────┘
```

- **顶部导航栏**：Logo、产品名称、主题切换按钮
- **对话消息区**：用户消息右对齐气泡，AI 消息左对齐气泡；AI 返回结果以资源卡片网格展示
- **资源卡片**：缩略图/图标 + 名称 + 格式标签 + 文件大小 + [预览] [下载] 按钮
- **底部输入栏**：多行文本输入框 + 发送按钮，支持 Enter 发送
- **预览弹窗**：点击预览弹出模态框，图片放大查看、视频内嵌播放、文档在线预览、链接展示摘要

**页面 2：资源预览模态框**

- 全屏/半屏模态框，居中展示资源内容
- 图片：自适应缩放，支持原图查看
- 视频：内置播放器，支持播放/暂停/进度条
- 文档：PDF/Word/Excel 在线预览（PDF 直接渲染，Office 文档转为预览）
- 链接：展示标题、描述、缩略图，点击"访问原链接"跳转新标签页

## 4. 待确认问题（Open Questions）

| 编号 | 问题 | 影响范围 | 建议 |
|------|------|----------|------|
| Q1 | **AI 对话的智能程度要求**：是否需要接入大语言模型（如 GPT/Claude）进行语义理解，还是基于关键词匹配 + 向量检索即可？ | 影响架构选型与成本 | 建议 P0 采用关键词 + 标签匹配 + 简单语义检索，P2 升级为 LLM 多轮对话 |
| Q2 | **资源存储方案**：文件存储使用本地磁盘、对象存储（如 S3/OSS）还是数据库 BLOB？ | 影响部署架构与成本 | 建议使用对象存储或本地磁盘 + CDN，便于扩展 |
| Q3 | **管理员账号体系**：是单管理员（环境变量配置）还是多管理员（数据库管理）？是否需要角色权限？ | 影响后台设计复杂度 | 建议 P0 单管理员（环境变量配置账号密码），P2 扩展多管理员 |
| Q4 | **部署平台偏好**：是否有偏好的部署平台（Vercel / 自建服务器 / 云服务器）？ | 影响技术选型与交付方式 | 建议 Next.js + Vercel 一键部署，或 Docker + 云服务器 |
| Q5 | **资源容量预期**：预计资源库的文件总数量和总大小量级？ | 影响存储方案与性能设计 | 需用户确认，以评估存储与检索性能需求 |
| Q6 | **前台是否需要移动端适配**：是否要求响应式设计适配手机/平板访问？ | 影响前端开发工作量 | 建议默认响应式适配，确保移动端可用 |
| Q7 | **资源安全与访问控制**：前台资源是否完全公开？是否需要防直链下载/水印等保护？ | 影响后端接口设计 | 建议 P0 完全公开访问，P2 可考虑访问控制 |

## 5. 技术选型建议（供架构师参考，非强制）

| 维度 | 建议 | 理由 |
|------|------|------|
| 前端框架 | Next.js (React) | 全栈一体化，SSR/SSG 支持，便于 SEO 与部署 |
| UI 组件库 | Tailwind CSS + shadcn/ui 或 MUI | 快速构建美观一致的界面 |
| 后端 | Next.js API Routes / 独立 Node 服务 | 与前端一体化部署，降低运维复杂度 |
| 数据库 | PostgreSQL / SQLite | PostgreSQL 适合生产环境，SQLite 适合轻量部署 |
| 文件存储 | 对象存储（S3 兼容）/ 本地磁盘 | 视部署平台而定 |
| AI 检索 | 关键词匹配 + 标签过滤 + 向量检索（可选） | P0 轻量方案，P2 可接入 LLM |

> **注**：以上仅为产品层面的技术建议，最终技术选型由架构师根据实际需求决定。
